# ai-work-result-evaluation

Этот навык применяется для проверки качества результата ИИ перед его принятием,
публикацией или повторным использованием.

## Обязательный порядок

1. Определи тип результата (правило, документ, шаблон, инструкция, ответ).
2. Укажи критерии проверки, метрику и порог пригодности.
3. Сравни с базовой линией, если она существует.
4. Прогони проверочные случаи, включая отрицательные.
5. Принять решение: `accept`, `revise`, `reject`, `needs_human_decision`.
6. Зафиксировать, какие провалы переходят в постоянный набор проверок.

## Что должно присутствовать

- список проверяемых сценариев;
- причина прохождения или падения каждого сценария;
- ссылка на базовую линию;
- решение по дальнейшему использованию результата.

## Границы

- Не заменяет проверку применимости задачи, это второй этап после `ai-application-check`.
- Не применим к разовым черновикам без значимых последствий.
