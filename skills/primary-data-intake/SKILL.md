# Навык: приём первичных данных

## Назначение

Этот навык задаёт порядок включения нового материала в корпус знаний проекта в
качестве первичных данных.

Навык нужен для того, чтобы новый источник попадал в проект не как произвольный
файл, а как учтённый элемент корпуса с понятным происхождением, статусом и
ограничениями.

## Когда применять навык

Навык применяется, если нужно:

- принять новый источник в проект;
- сохранить материал в максимально исходном виде;
- отделить первичный материал от его обработки;
- зафиксировать происхождение и ограничения источника;
- понять, можно ли использовать материал как основание для дальнейшей работы.

## Входные данные

Для применения навыка должны быть известны настолько, насколько это возможно:

- сам материал или доступ к нему;
- источник происхождения;
- способ получения;
- дата или момент получения;
- предполагаемая роль материала в проекте;
- известная лицензия или иные условия использования;
- видимость проекта по `AGENTS.md`;
- ограничения по полноте, качеству, доступу или доверию.

Если это неизвестно, нехватка данных должна быть зафиксирована явно.

## Порядок применения

### 1. Определить видимость проекта

Перед выбором способа хранения источника проверь `AGENTS.md` проекта.

Если в нём явно указано, что проект закрытый, применяй правила закрытого
проекта.

Если в нём явно указано, что проект публичный, применяй правила публичного
проекта.

Если видимость проекта не указана, считай проект публичным.

### 2. Идентифицировать материал

Нужно зафиксировать:

- что именно получено;
- в какой форме это получено;
- является ли это исходным материалом или уже производным представлением;
- почему этот материал относится к проекту.

### 3. Зафиксировать происхождение

Нужно указать:

- откуда получен материал;
- каким способом он получен;
- когда он получен;
- какая лицензия или какие условия использования известны;
- есть ли признаки неполноты, повреждения или сомнительного происхождения.

Если происхождение неясно, материал нельзя считать надёжным основанием для
значимых выводов.

Если источник используется в публичном проекте, а лицензия или условия
использования неясны, это должно считаться отдельным ограничением при приёме.

### 4. Сохранить материал в максимально исходном виде

Материал должен быть включён в корпус так, чтобы:

- не терялось исходное содержание;
- не происходила тихая правка ради удобства;
- не смешивались исходный материал и его последующая обработка;
- можно было повторно обратиться к первичному состоянию.

Для закрытого проекта сохраняй исходный материал в максимально исходной форме в
корпусе первичных данных.

Для публичного проекта сначала проверь лицензию и условия использования
источника.

Если лицензия разрешает нужное использование, сохраняй исходный материал в Git и
фиксируй выполнение требований лицензии.

Если лицензия запрещает нужное использование, сохраняй исходный материал только
в локальной неотслеживаемой области, исключай ранние стадии обработки из Git и
добавляй в Git только допустимые производные материалы: подробный пересказ,
выводы, факты, ссылки, метаданные, хэши и ограничения использования.

### 5. Зафиксировать ограничения

Нужно явно отметить:

- достаточно ли материал полон;
- можно ли ему доверять без оговорок;
- есть ли ограничения доступа;
- есть ли ограничения лицензии или публикации;
- можно ли хранить исходный материал в Git;
- с какой стадии обработки данные можно отслеживать в Git;
- нужны ли дополнительные действия по проверке, очистке, нормализации или
  расшифровке;
- можно ли уже использовать материал для следующего шага.

### 6. Отделить первичный материал от производных слоёв

Если для работы нужны:

- нормализация;
- расшифровка;
- структурирование;
- извлечение утверждений;
- индексирование;
- иные производные операции,

они должны выполняться отдельно от приёма первичного материала.

### 7. Зафиксировать решение по приёму

По итогам приёма должно быть понятно:

- принят ли материал в корпус;
- принят ли он как первичный или как производный;
- какие ограничения у него есть;
- можно ли хранить его исходное состояние в Git;
- можно ли использовать его дальше;
- что ещё нужно сделать до следующего шага.

## Результат применения навыка

Результатом должен быть элемент корпуса, для которого можно ответить на вопросы:

- что это за материал;
- почему он включён в проект;
- каково его происхождение;
- первичный он или производный;
- в каком состоянии он находится;
- какие ограничения нужно учитывать;
- можно ли на него опираться дальше.

## Что считается ошибкой применения навыка

- источник принят без фиксации происхождения;
- источник принят для публичного использования без фиксации известных условий
  использования;
- видимость проекта не проверена и не применено правило публичного проекта по
  умолчанию;
- исходный материал источника сохранён в Git публичного проекта вопреки
  лицензионному запрету;
- пересказ принят вместо исходного материала;
- производный слой сохранён как будто это первичный материал;
- материал переписан ради удобства до фиксации исходного состояния;
- ограничения источника не отмечены;
- материал признан пригодным для дальнейших выводов без достаточного основания.

## Связь с другими элементами продукта

Этот навык должен применяться вместе:

- с принципом корпуса первичных данных;
- с ролью `source-inventory`;
- с контрольным слоем, если приём источника влияет на значимые решения проекта.
