# Разработка продукта

## Установка и обновление зависимостей

```shell
apm install --target agent-skills --frozen
npm install
```

Чтобы обновить закреплённые версии APM-зависимостей и lockfile, выполните:

```shell
apm deps update --target agent-skills
```

## Выпуск версии

Перед тегом новой версии обновите `CHANGELOG.md`:

1. Перенесите записи из раздела «Невыпущено» в раздел новой версии с датой.
2. Если выпуск требует действий от проектов-потребителей (переименование или
   удаление навыка, агента или правила, изменение структуры проекта или
   корпуса), добавьте в раздел версии подраздел «Действия при обновлении» с
   привязкой к примитиву. Если действий не требуется, подраздел не добавляйте.

Раздел «Действия при обновлении» читает ветка обновления версии навыка
`ait-setup` в проекте-потребителе: по нему агент проводит человека по пересмотру
проекта при переходе на новую версию продукта.

Журнал изменений самого продукта `ai-dev-team` и журналы изменений
проектов-потребителей — разные поверхности. Продуктовый `CHANGELOG.md`
обновляется в этом репозитории перед выпуском версии. Журнал целевого проекта
сопровождается через `ait-changelog` и меняется только по правилам самого
проекта.
