# Корпус знаний проекта

Корпус знаний проекта, или просто «корпус знаний», это набор источников и
производных материалов, на которые агенты должны опираться в работе.

## Что входит в корпус знаний

В корпус знаний входят материалы, которые проект считает опорными для:

- выполнения задач;
- проверки изменений;
- подготовки новых результатов;
- обновления производных документов по изменениям источников.

## Расположение по умолчанию

По умолчанию файлы корпуса знаний хранятся в папке `knowledge/` в корне целевого
проекта.

Если проект не задал отдельное правило, агент должен считать папку `knowledge/`
основным местом для таких материалов.

## Как изменить расположение

Целевой проект может задать другое расположение корпуса знаний в машинных
инструкциях проекта.

Для этого в машинных инструкциях нужно явно указать, где хранятся файлы корпуса
знаний. Формулировка должна быть однозначной и содержать относительный путь от
корня проекта.

Если проект поддерживает несколько клиентов, это правило должно жить в
переносимом источнике инструкций, а не только в файле конкретного клиента.

Пример:

```md
Файлы корпуса знаний хранятся в папке `docs/knowledge`.
```

Если такое правило задано, агент должен использовать указанный путь вместо
`knowledge/`.

## Как с ним работать

Основная работа с корпусом знаний состоит в управлении источниками.

Обычно нужно:

1. Добавлять новые источники, если они должны использоваться агентами.
2. Проверять, изменились ли подключённые источники.
3. Разбирать результаты проверки и решать, нужно ли обновлять производные
   материалы.
4. Удалять или заменять источники, если они больше не должны входить в корпус
   знаний.
