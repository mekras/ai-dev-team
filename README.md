# Команда агентной разработки ПО

Команда агентов ИИ, которую можно подключить к проекту по разработке ПО.
Включает в себя необходимые роли навыки и правила работы, проверки качества и
согласованный порядок принятия решений. Обеспечивает стабильный уровень качества
за счёт системного применения накопленных практик и снижает риски, вызванные
человеческим фактором.

## Подключение к проекту

Для подключения необходим
[Agent Package Manager](https://github.com/microsoft/apm).

Выберите клиент и выполните команду в корне целевого проекта.

Для Claude:

```shell
apm install mekras/ai-dev-team#^0.9.0 --target claude
```

Для Codex:

```shell
apm install mekras/ai-dev-team#^0.9.0 --target codex
```

APM добавит продукт в зависимости текущего проекта, установит локальную копию
продукта и его зависимости, затем развернёт навыки для выбранного клиента. Не
опускайте `--target`: проект нужно подключать явно либо для `claude`, либо для
`codex`.

После установки откройте выбранный инструмент для общения с агентами в корне
проекта и дайте команду:

```text
Настрой проект.
```

Навык [ait-setup](.apm/skills/ait-setup/SKILL.md) проверит текущие файлы
проекта, сохранит существующие локальные правила из `AGENTS.md` или `CLAUDE.md`
и обновит клиентские инструкции без потери данных. Если у проекта уже есть
журнал изменений, настройка предложит закрепить его сопровождение. Если журнала
нет, она может предложить создать `CHANGELOG.md`.

Проверкой первичной установки считается наличие продукта в зависимостях проекта
и доступность навыка `ait-setup` в выбранном клиенте. Проверкой настройки
считается успешное завершение команды «Настрой проект».

## Обновление продукта

Когда выходит новая версия продукта, обновите её в проекте и пересмотрите проект
под изменения.

Сначала установите новую версию тем же клиентом, что и при подключении.

Для Claude:

```shell
apm install mekras/ai-dev-team#^0.9.0 --target claude
```

Для Codex:

```shell
apm install mekras/ai-dev-team#^0.9.0 --target codex
```

Затем в корне проекта дайте агенту команду:

```text
Обнови проект под новую версию ai-dev-team.
```

Навык `ait-setup` определит целевую версию по установленному продукту, а уже
применённую версию — по файлу `.ai-dev-team-state.yml` в корне вашего проекта.
Затем он прочитает журнал изменений продукта и проведёт вас по разделу «Действия
при обновлении»: по каждому изменению, которое требует правок в вашем проекте, с
подтверждением каждого шага. Эта команда не должна повторно обновлять
зависимости APM: новая версия уже установлена, дальше нужен пересмотр самого
проекта. Если выпуск не требует действий в проекте, навык сообщит об этом и
обновит файл состояния.

Что изменилось между версиями, описано в [журнале изменений](CHANGELOG.md).

Для журналов изменений самих проектов-потребителей используется навык
`ait-changelog`: он сохраняет формат проекта, пишет только внешне значимые
изменения и добавляет действия при обновлении, когда они действительно нужны.

## Разработка

### Зависимости разработки продукта

При разработке самого `ai-dev-team` зависимые коллекции навыков подключаются
через `APM`.

В корне этого репозитория выполните:

```shell
apm install --target agent-skills --frozen
npm install
```

Если нужно обновить закреплённые зависимости и lockfile, выполните:

```shell
apm deps update --target agent-skills
```
