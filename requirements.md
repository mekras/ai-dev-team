# Требования к продукту

## Определения

- _Продукт_ — разрабатываемый продукт. Располагается в папке [team](team).
- _Проект_ — этот проект по разработке _продукта_.
- _Целевой проект_ — сторонний проект, в котором будет или уже используется
  _продукт_.
- _Заказчик_ — человек, управляющий разработкой _целевого проекта_.

## Требования к проекту и разработке продукта

Приоритетами в разработке продукта являются надёжность, качество и эффективность
его применения в целевых проектах. Выполнение всех прочих требований и задач не
должно негативно влиять на эти качества.

- Проект должен рассматриваться как частный случай целевого проекта, т. е. он
  должен подключать и использовать продукт так же, как это предусмотрено для
  целевых проектов.

## Требования к способам использования продукта

- Продукт должен требовать от заказчика наименьшее возможное количество действий
  для подключения к целевому проекту.
- Одну копию продукта должно быть можно использовать для нескольких целевых
  продуктов.
- Изменения внутренней структуры проекта не должны требовать изменений в целевых
  проектах.
- Подключение целевого проекта должно задаваться как логический контракт
  (`ai-dev-team@<канал>`), а не как путь к локальным файлам продукта.
- Способ подключения не должен требовать сохранять в файлах под управлением VCS
  специфических для конкретной системы данных (например, файловых путей).
- Цели поддерживаемых инструментов должны задаваться как проектное значение по
  умолчанию, а не повторяться в каждой команде APM.

## Ограничения

- Продукт должен где возможно использовать универсальные стандарты, а не
  специфичные для конкретного инструмента решения. Например, требовать
  использовать `AGENTS.md`, а не `CLAUDE.md`.
- Продукт должен поддерживать следующие инструменты:
  - Claude
  - Codex
