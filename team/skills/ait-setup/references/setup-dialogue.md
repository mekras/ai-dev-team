# Диалог настройки проекта

Используй эту справку, когда `ait-setup` должен собрать сведения перед созданием
или обновлением стартовых файлов проекта.

## Минимальные вопросы

Задавай вопросы по одному. Один ход агента должен содержать только один
обязательный вопрос к человеку. Исключения допустимы только если человек прямо
просит анкету, просит перечислить все недостающие сведения или уже дал данные
для нескольких решений.

Для большинства проектов нужно выяснить:

- название проекта;
- одно предложение о назначении проекта;
- кто основной пользователь или читатель README;
- публичный проект или закрытый;
- кто владелец или автор для лицензии;
- какая лицензия выбрана;
- используется ли Git;
- должен ли проект использовать `ai-dev-team` через APM.

Если пользователь уже дал часть сведений, не спрашивай их повторно.

## Порядок диалога

Начинай с самого блокирующего решения и переходи дальше только после ответа
человека. Рекомендуемый порядок:

1. Название и краткое назначение проекта.
2. Публичный проект или закрытый.
3. Используется ли Git.
4. Должен ли проект использовать `ai-dev-team` через APM.
5. Кто владелец или автор для лицензии.
6. Какая лицензия выбрана или нужно пока не выбирать лицензию.

Если один ответ человека закрывает несколько пунктов, отметь их как известные и
задай следующий один вопрос.

Не задавай в одном сообщении список вопросов вида «уточните название, лицензию и
Git». Это нарушает диалоговый режим `ait-setup`.

## Лицензия

Не выбирай лицензию за человека.

Допустимые безопасные варианты вопроса:

- «Какая лицензия нужна: MIT, Apache-2.0, GPL-3.0, проприетарная или пока не
  выбирать?»
- «Кого указать владельцем прав в `LICENSE`?»

Если лицензия не выбрана, не создавай файл с текстом открытой лицензии. Можно
предложить отложить `LICENSE` или создать файл с явным статусом «лицензия не
выбрана», только если человек подтвердил такой вариант.

## README

Для минимального `README.md` нужны:

- название проекта;
- краткое назначение;
- первое практическое действие пользователя;
- команда установки, запуска или подключения, если она уже известна.

Если команда неизвестна, не выдумывай её. Оставь короткий открытый вопрос или
раздел «Статус», если человек разрешил черновой README.

## AGENTS.md

Для минимального `AGENTS.md` нужны:

- строка подключения вида «Проект использует `ai-dev-team`.», если проект должен
  использовать `ai-dev-team`;
- видимость проекта: публичный или закрытый;
- расположение корпуса знаний, если оно отличается от `knowledge/`;
- правила языка, форматирования, Git и проверки, если они уже известны.

Версия продукта и его локальная копия определяются APM-зависимостью целевого
проекта, а не строкой подключения в `AGENTS.md`.

## .gitignore

Создавай `.gitignore`, только если проект использует Git.

Если стек неизвестен, минимально допустимы:

- `.DS_Store`;
- временные редакторские файлы;
- `.env`;
- `.env.*`;
- `node_modules/`, если проект на Node.js;
- `__pycache__/` и `.venv/`, если проект на Python.

Не добавляй в `.gitignore` путь приватного локального слоя, если человек хочет
скрыть сам факт его существования. Для Git в таком случае используй
`.git/info/exclude` через `private-project-knowledge`.

## Подагенты

После стартовых файлов предложи рациональное использование подагентов:

- быстрая модель для узкого поиска и извлечения фактов;
- основная модель для обычной работы и финального ответа;
- сильная модель для редкой проверки ответственных решений;
- строгие границы делегирования и проверка результата родительским агентом.

Не внедряй конфигурацию подагентов без подтверждения человека.

## Дополнительные стартовые файлы

Можно предложить, но не создавать без подтверждения:

- `.editorconfig`;
- `docs/concept.md`;
- `requirements.md`;
- начальный `knowledge/README.md`;
- приватный локальный слой знаний агента.
