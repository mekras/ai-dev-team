# Проверка: README коллекции навыков APM

## Ситуация

Агенту поручили переписать главный `README.md` проекта, который является
коллекцией навыков APM.

В проекте есть:

- каталог навыков;
- команды проверки триггеров навыков;
- локальные пути разработки;
- похожий пользовательский `README.md` в соседнем проекте.

## Ожидаемое поведение

Агент должен:

- определить основного читателя как пользователя коллекции навыков;
- описать, зачем устанавливать коллекцию;
- дать установку через `apm install -g github.com/<owner>/<repo>#<branch>`;
- указать обновление через `apm deps update -g`;
- объяснить, что после установки навыки доступны агенту как локальные навыки;
- перечислить навыки ниже как состав коллекции;
- вынести проверки триггеров и локальные пути в раздел разработки или не
  включать их.

## Ошибочное поведение

Агент не должен:

- начинать `README.md` со структуры репозитория;
- помещать команды проверки триггеров в пользовательский быстрый старт;
- использовать локальный путь как основной способ установки для внешнего
  пользователя;
- подменять назначение коллекции перечислением внутренних случаев;
- использовать заголовок «Что даёт коллекция» для списка каталогов или служебных
  файлов.

## Критерий прохождения

Новый пользователь за первую минуту понимает:

- что это коллекция навыков;
- зачем её устанавливать;
- какой командой установить или обновить коллекцию;
- что изменится для агента после установки;
- где находятся детали для сопровождающего проекта.
