# Навык: нормализация первичных данных

## Назначение

Этот навык задаёт порядок подготовки нормализованного представления первичных
данных без изменения их статуса как исходного материала.

Навык нужен для того, чтобы проект мог переходить от хранения источников к
дальнейшей обработке без потери происхождения, структуры и ограничений.

## Когда применять навык

Навык применяется, если нужно:

- подготовить первичные данные к чтению, поиску, сравнению или извлечению
  утверждений;
- убрать шум форматирования и технические артефакты;
- создать единообразное представление набора источников;
- явно отделить исходный слой от слоя обработки.

## Входные данные

Для применения навыка должны быть доступны:

- первичный материал;
- сведения о его происхождении;
- известные ограничения источника;
- понимание, для какого следующего шага готовится нормализованный слой.

Если этих данных недостаточно, ограничение должно быть зафиксировано явно.

## Порядок применения

### 1. Определить цель нормализации

Нужно зафиксировать:

- какой материал нормализуется;
- для какого следующего шага это делается;
- какие преобразования допустимы;
- какие преобразования недопустимы.

### 2. Сохранить связь с первичным материалом

До начала нормализации должно быть понятно:

- на какой первичный материал опирается работа;
- как будет сохраняться ссылка на исходник;
- как будет отличаться нормализованный слой от исходного.

Если эта связь теряется, нормализация выполнена неправильно.

### 3. Выполнить допустимые преобразования

Допустимы преобразования, которые:

- убирают технический шум;
- приводят формат к последовательному виду;
- улучшают читаемость и сопоставимость;
- делают структуру материала пригодной для следующего шага;
- не меняют смысл исходного материала.

### 4. Явно отметить ограничения

Нужно отдельно отметить:

- неполные места;
- повреждённые фрагменты;
- сомнительные чтения;
- неоднозначные участки;
- места, где автоматическая обработка могла быть неточной.

Такие места должны оставаться явными ограничениями нормализованного слоя.

### 5. Отделить нормализацию от последующих этапов

На этом этапе результатом должна оставаться именно нормализация материала.

Поэтому следующие действия относятся уже к другим шагам:

- извлечение утверждений;
- проектные выводы;
- интерпретация намерений автора;
- формулирование требований;
- выбор смысла спорного места без явной фиксации ограничения.

Если это уже требуется, работа должна переходить к следующему навыку или
следующей роли.

### 6. Зафиксировать результат

По итогам должно быть понятно:

- какой материал был нормализован;
- что именно было изменено на уровне формы;
- что осталось ограничением;
- можно ли использовать нормализованный слой дальше;
- какой следующий этап допустим.

## Результат применения навыка

Результатом должен быть нормализованный слой, который:

- основан на конкретном первичном материале;
- не подменяет исходник;
- пригоден для следующего шага обработки;
- сохраняет ограничения и неоднозначности в явном виде.

## Что считается ошибкой применения навыка

- нормализованный слой не связан с первичным материалом;
- первичный материал переписан вместо создания производного слоя;
- форматное преобразование изменило смысл;
- спорные места были сглажены без отметки;
- нормализация незаметно перешла в интерпретацию или выводы;
- следующий шаг невозможен, потому что результат остался неструктурированным.

## Связь с другими элементами продукта

Этот навык должен применяться вместе:

- с принципом корпуса первичных данных;
- с навыком приёма первичных данных в корпус;
- с ролью `source-inventory`;
- с ролью нормализатора данных;
- с контрольным слоем, если качество нормализации влияет на значимые решения
  проекта.
