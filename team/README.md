# Команда агентной разработки ПО

Переиспользуемая команда агентов ИИ для разработки ПО.

1. **Системный подход**. Не просто набор навыков и правил, а целостная команда,
   управляющая всем процессом разработки от формулирования концепции до выпуска.
2. **Контроль применения ИИ**. Фундамент решения — особый слой правил и навыков,
   контролирующий правильное и разумное использование ИИ в проекте.
3. **Корпус знаний проекта**. Все исходные данные проекта сохраняются в
   неизменном виде, чтобы их можно было многократно обрабатывать без потери
   смысла при любых изменениях в проекте.
4. **Проектные роли**. Полноценная команда, включающая менеджера проекта,
   аналитика, технического писателя, программиста, тестировщика и другие роли.

По сути — это команда, которой можно поручить работу с вашим проектом.

## Использование

### Подключение к проекту

Клонируйте продукт в любую папку **вне** вашего проекта:

```shell
git clone https://github.com/mekras/ai-dev-team.git
```

Если в вашем проекте ещё нет файла [AGENTS.md](https://agents.md), создайте его.

В файл `AGENTS.md` добавьте:

```markdown
Проект использует `ai-dev-team@stable`.
```

Это подключит все правила, роли и навыки команды к вашему проекту.

Содержимое `AGENTS.md` целевого проекта должно быть только декларированием
идентификатора и не должно содержать локальные пути к продукту.
