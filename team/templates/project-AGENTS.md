# Инструкции проекта для агентов

## Назначение проекта

Опишите, что делает этот проект, для кого он нужен и какие границы у работы
агента.

## Подключённая команда

Проект использует продукт `ai-dev-team`.

Идентификатор подключения:

```text
ai-dev-team@stable
```

Агент должен использовать из продукта:

- фундаментальные принципы из `foundation/`;
- слой управления ИИ из `ai-control/`;
- роли из `roles/`;
- индекс `skills/INDEX.md` и навыки из `skills/`;
- правила работы с первичными данными и источниками.

## Локальный слой проекта

Локальные правила проекта задаются в этом файле и в других проектных документах.

К локальному слою относятся:

- цели и границы проекта;
- приватные знания и ограничения;
- проектные источники данных;
- проектные решения, требования и документация;
- исключения, которые не относятся к переносимому ядру продукта.

## Контроль применения ИИ

Перед существенными задачами агент применяет слой управления ИИ подключённого
продукта.

Существенной считается задача, если она:

- меняет правила проекта;
- меняет требования, решения или документацию;
- добавляет или изменяет источники данных;
- создаёт вывод, который может стать основанием для решения;
- требует выбора роли, навыка или процедуры.

Результат проверки фиксируется здесь:

```text
УКАЖИТЕ_МЕСТО_ДЛЯ_ПРОВЕРОК
```

## Корпус знаний проекта

Первичные данные проекта (часть корпуса знаний) хранятся здесь:

```text
УКАЖИТЕ_МЕСТО_ДЛЯ_ПЕРВИЧНЫХ_ДАННЫХ
```

Производные материалы должны храниться отдельно от первичных данных.

## Обновление продукта

Переносимые файлы продукта не правятся как локальные правила проекта.

Локальные изменения проекта фиксируются в локальном слое. Если локальная
практика может быть полезна другим проектам, она оформляется как кандидат на
перенос в продукт и рассматривается отдельно.
