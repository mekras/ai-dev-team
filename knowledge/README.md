# Корпус знаний `ai-dev-team`

Этот каталог содержит корпус знаний проекта и задаёт его локальный договор
структуры.

## Слои корпуса

- `inventory/` — карточки учёта источников по коду источника.
- `primary/` — первичные артефакты и паспорта источников.
- `normalized/` — нормализованные представления первичных материалов.
- `statements/` — проверяемые утверждения, извлечённые из источников.
- `reports/source-impact/` — отчёты о влиянии источников на навыки, документы и
  другие файлы проекта.
- `source-attribution.md` — правила атрибуции и ссылок на источники.

## Правило соответствия

Для одного и того же кода источника ожидается согласованная цепочка между
`inventory/`, `primary/`, `normalized/` и `statements/`.

Если проект заявляет использование корпуса знаний, агент должен проверять
наличие и согласованность этих слоёв перед значимыми изменениями в коде,
документации, правилах, требованиях и архитектуре.

Если структура корпуса меняется, сначала обновляй этот файл и связанные правила
агента, а затем проверяй корпус через `kc-validation`.
