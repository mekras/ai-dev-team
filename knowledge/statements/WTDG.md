# Извлечённые утверждения: `WTDG`

## Основание

- Источник учёта: `knowledge/inventory/WTDG.md`
- Нормализованный слой: `knowledge/normalized/WTDG/source.md`
- Первичный снимок обзорной страницы: `knowledge/primary/WTDG/index.html`
- Первичные снимки внутренних страниц: `knowledge/primary/WTDG/pages`
- Вспомогательные очищенные копии: `knowledge/normalized/WTDG/legacy-source.md`,
  `knowledge/normalized/WTDG/pages`
- Атрибуция источника: `knowledge/source-attribution.md`

## Назначение

Этот документ фиксирует утверждения по корпусу `WTDG`, пригодные для дальнейшей
работы над каркасом.

Утверждения ниже относятся к одному источнику: разделу `guide/` сайта
`Write the Docs` вместе с его внутренними страницами, полученными из обзорной
страницы.

## Что агент уже может использовать

- как основание для роли технического писателя:
  - документация как часть продукта;
  - технический писатель как самостоятельная роль, а не только редактор;
  - работа от аудитории, целей, пользовательского контекста и структуры знания;
- как основание для будущих навыков:
  - раннее документирование;
  - минимальный стартовый комплект документации;
  - стайлгайды;
  - доступность;
  - снижение предвзятости;
  - UX writing;
  - выбор инструментов;
  - документация API;
- как основание для проектных процедур:
  - начинать с понятной проблемы;
  - упрощать вход в документацию через шаблоны и иерархию;
  - включать регулярную обратную связь и долгосрочное сопровождение;
- как ограничение:
  - различать страницы с собственными рекомендациями и страницы-каталоги;
  - не превращать подборку внешних ресурсов в прямое правило проекта без
    отдельной проверки;
  - учитывать лицензионные ограничения при публичных производных материалах.

## Пригодность к использованию

Этот источник уже пригоден к использованию как единый рабочий корпус для:

- развития роли технического писателя;
- разработки навыков технического письма, стайлгайдов и документационных
  процедур;
- выбора дальнейших тем для углубления без дробления источника на отдельные
  внешние источники.

Источник уже даёт содержательные основания для проектных решений, а не только
обзорную навигацию. При этом при точечной опоре на конкретную мысль агент всё
равно должен ссылаться на конкретную страницу внутри корпуса.

## Ограничения

- Не все страницы корпуса одинаково содержательны: часть из них даёт
  рекомендации, часть в основном собирает внешние ссылки.
- Источник не должен автоматически превращаться в решения для каркаса без
  отдельного анализа.
- При публичном использовании производных материалов требуется учёт условий
  `CC BY-NC-SA 4.0`.
