## Назначение

Документ фиксирует утверждения по `OWASP SAMM v2.0`, достаточные для работы роли
`security-engineer` с оценкой и улучшением программы безопасной разработки.
Утверждения фиксируют содержание модели, а не решения репозитория. Опора —
локальная текстовая выгрузка PDF. Все утверждения имеют статус
`ready_for_review`. Лицензия источника — `CC BY-SA 4.0` (требуется атрибуция).

## Структура модели

### SAMM-001. Назначение модели

- Утверждение: SAMM — открытая модель оценки и улучшения программы безопасной
  разработки ПО; она технологически нейтральна и применяется итеративно (оценка
  → цели → план улучшений).
- Фрагмент источника: вводный раздел.
- Область применения: специалист по безопасности, project-manager.

### SAMM-002. Пять бизнес-функций

- Утверждение: модель построена вокруг пяти бизнес-функций: Governance, Design,
  Implementation, Verification, Operations.
- Фрагмент источника: оглавление и обзор практик.
- Область применения: специалист по безопасности.

### SAMM-003. Пятнадцать практик безопасности

- Утверждение: в каждой бизнес-функции по три практики безопасности (всего 15);
  каждая практика имеет два потока (stream A/B) и три уровня зрелости.
- Фрагмент источника: обзор практик и модель зрелости.
- Область применения: специалист по безопасности.

## Практики по бизнес-функциям

### SAMM-004. Governance

- Утверждение: функция Governance включает практики Strategy & Metrics, Policy &
  Compliance, Education & Guidance.
- Фрагмент источника: раздел Governance.
- Область применения: project-manager, специалист по безопасности.

### SAMM-005. Design

- Утверждение: функция Design включает практики Threat Assessment, Security
  Requirements, Security Architecture; Threat Assessment охватывает
  моделирование угроз.
- Фрагмент источника: раздел Design.
- Область применения: специалист по безопасности, архитектор, аналитик.

### SAMM-006. Implementation

- Утверждение: функция Implementation включает практики Secure Build, Secure
  Deployment, Defect Management.
- Фрагмент источника: раздел Implementation.
- Область применения: разработчик, специалист по безопасности.

### SAMM-007. Verification

- Утверждение: функция Verification включает практики Architecture Assessment,
  Requirements-driven Testing, Security Testing.
- Фрагмент источника: раздел Verification.
- Область применения: специалист по безопасности, роль проверки результата.

### SAMM-008. Operations

- Утверждение: функция Operations включает практики Incident Management,
  Environment Management, Operational Management.
- Фрагмент источника: раздел Operations.
- Область применения: специалист по безопасности.

## Модель зрелости

### SAMM-009. Три уровня зрелости

- Утверждение: каждая практика имеет три уровня зрелости с растущей строгостью и
  охватом; уровни описаны через активности, вопросы оценки и ожидаемые
  результаты, что позволяет измерять прогресс.
- Фрагмент источника: модель зрелости.
- Область применения: специалист по безопасности, project-manager.

### SAMM-010. Итеративное применение

- Утверждение: SAMM применяется итеративно: оценить текущее состояние, поставить
  достижимые цели по уровням и реализовать план улучшений, повторяя цикл.
- Фрагмент источника: руководство по применению модели.
- Область применения: специалист по безопасности, project-manager.
