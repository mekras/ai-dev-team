# Building effective agents (нормализованное представление)

## URL

- `https://www.anthropic.com/engineering/building-effective-agents`

## Ключевые тезисы

- Anthropic различает `workflows` как заранее заданные кодовые траектории и
  `agents` как системы, где модель сама направляет процесс и выбор инструментов.
- Для LLM-приложений рекомендуется начинать с самого простого решения и
  добавлять агентную сложность только если она даёт измеримый выигрыш.
- Для хорошо определённых задач предпочтительнее workflow-паттерны, а для
  открытых задач с непредсказуемым числом шагов подходят агенты.
- Базовый строительный блок агентной системы — augmented LLM с retrieval,
  tools и memory.
- Фреймворки ускоряют старт, но могут скрывать реальные prompts и responses;
  при использовании фреймворка важно понимать нижележащий код и не наращивать
  абстракции без пользы.
- В качестве типовых workflow-паттернов перечислены prompt chaining, routing,
  parallelization, orchestrator-workers и evaluator-optimizer.
- Для production-агентов важны прозрачность, простота дизайна и тщательно
  спроектированный agent-computer interface через инструменты.
