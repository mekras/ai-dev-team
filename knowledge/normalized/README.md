# Нормализованный слой корпуса знаний

Этот каталог содержит нормализованные представления первичных данных проекта.

## Назначение

Нормализованный слой подготавливает компактное и последовательное представление
источников для последующего извлечения утверждений. Он не заменяет исходные
снимки первичных данных, а приводит их к форме, удобной для обзора и
аналитической работы.

## Структура

### Источники с многофайловой структурой

Для источников с несколькими страницами нормализованный слой организуется как
каталог с файлом обзора и каталогом страниц:

```text
normalized/
├── ASKL/
│   ├── source.md          # Обзорная страница
│   └── pages/             # Нормализованные страницы
│       ├── best-practices/
│       ├── evaluating-skills/
│       └── optimizing-descriptions/
├── PPRX/
│   ├── source.md
│   └── pages/
├── WTDG/
│   └── pages/
└── XEM7/
    └── pages/
```

### Источники с единой структурой

Для источников с единым содержательным ядром (например, снимок commit с
вложенной структурой) нормализованное представление может быть единым файлом:

- `HWD.md` — нормализованное представление структуры HWD (снимок commit)

### Устаревшие файлы

Некоторые файлы отражают предыдущие версии процедуры нормализации:

- `WTDG-source.md` — исходная обзорная страница, создана до
  разделения на каталог `pages/`. Сохранена для истории, актуальная
  информация дублируется в `WTDG.md`.

## Связь с другими слоями

Нормализованный слой связан с:

- **Первичные данные** (`../primary/`) — исходные снимки источников
- **Утверждения** (`../statements/`) — извлечённые факты, наблюдения, ограничения
- **Инвентарь** (`../inventory/`) — реестр источников с метаданными
- **Отчёты о влиянии** (`../reports/source-impact/`) — анализ влияния изменений

## Процедура нормализации

Нормализация выполняется по следующему порядку:

1. Получение первичных данных в `../primary/`
2. Создание описания источника в `../inventory/`
3. Нормализация структуры и содержания в этот каталог
4. Извлечение утверждений в `../statements/`
5. Фиксация влияния на производные документы

Подробнее см. навык `skills/primary-data-normalization/SKILL.md`.
