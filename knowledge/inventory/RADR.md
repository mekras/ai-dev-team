# Карточка источника: `RADR`

## Идентификатор

`RADR`

## Статус

- Состояние учёта: учтён
- Состояние получения: получен; полный PDF и текстовая выгрузка сохранены
  локально и исключены из Git
- Состояние пригодности: пригоден для дальнейшей работы с ограничениями

## Что это за источник

Локальная PDF-книга `REST API Design Rulebook` Марка Массе (Mark Massé),
изданная O’Reilly в `2012` году. Книга построена как свод именованных правил
проектирования веб-ориентированных REST API.

## Почему источник относится к проекту

Источник рассматривается как методическое основание для:

- проектирования идентификаторов ресурсов (URI) и архетипов ресурсов;
- проектирования взаимодействия по HTTP: методы и коды состояния;
- проектирования метаданных: заголовки представления и кэширования, медиатипы;
- проектирования представлений: JSON, гипермедиа (HATEOAS), формы ссылок, связей
  и ошибок;
- учёта потребностей клиентов: версионирование, безопасность, частичные ответы,
  кросс-доменный доступ.

## Что уже известно по источнику

- Книга формулирует около `80` именованных правил (`Rule:`) с модальностью must
  / should / may по шести главам.
- Степень соответствия REST оценивается по модели зрелости Леонарда Ричардсона
  (уровни URI, HTTP, Hypermedia).
- Автор вводит собственную рамку WRML (Web Resource Modeling Language) с
  медиатипом `application/wrml` и подключаемыми компонентами format и schema.
- Главы: введение и архитектура Веба, проектирование URI, взаимодействие по
  HTTP, метаданные, представления, потребности клиентов.

## Что можно делать дальше

- использовать обзорный нормализованный слой по структуре и правилам книги;
- извлекать краткие проверяемые утверждения по отдельным правилам и главам;
- использовать книгу как одно из оснований для навыка `software-architecture` в
  части проектирования веб-API.

## Что требует осторожности

- не публиковать в Git полный PDF, полную текстовую выгрузку и крупные дословные
  фрагменты;
- учитывать возраст книги (`2012`) при применении технических рекомендаций;
- не выдавать правила книги за принятые решения проекта;
- различать наблюдения из книги, интерпретации агента и локальные правила этого
  репозитория.
