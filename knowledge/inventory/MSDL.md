# Карточка источника: `MSDL`

## Идентификатор

`MSDL`

## Статус

- Состояние учёта: учтён
- Состояние получения: `external_reference` — локальная копия не хранится
- Состояние пригодности: пригоден

## Что это за источник

`Microsoft Security Development Lifecycle (SDL)` — методология встраивания
безопасности в разработку, выраженная набором практик по этапам ЖЦ.

## Почему источник относится к проекту

Отраслевой пример безопасного ЖЦ для навыков `security-audit` и
`security-threat-modeling`, в дополнение к предписывающим `SSDF`/`SAMM`.

## Основание в корпусе

- Паспорт: `knowledge/primary/MSDL/source.md`
- Нормализованный обзор: `knowledge/normalized/MSDL/source.md`
- Утверждения: `knowledge/statements/MSDL.md`

## Происхождение

- Издатель: Microsoft
- Канонический адрес: `https://www.microsoft.com/en-us/securityengineering/sdl`
- Дата включения в корпус: `2026-06-25`

## Тип источника

- Тип материала: внешний справочный источник (`external_reference`)
- Тип формы: методология (веб, проприетарно)

## Условия использования

- Лицензионный статус: материалы Microsoft защищены авторским правом; открытой
  лицензии нет.
- Допустимое использование: краткие сведения и ссылки без крупных дословных
  фрагментов.
- Атрибуция: `knowledge/source-attribution.md`.

## Что требует осторожности

- набор практик обновляется; сверять по адресу;
- проприетарно — не публиковать дословные фрагменты; тема пересекается с
  `TMDS`/`SSDF`/`SAMM`.
