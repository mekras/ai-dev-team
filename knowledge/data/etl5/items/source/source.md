# Источник: `ETL5`

## Статус

Источник принят в корпус знаний как локально сохранённая копия отчёта
`ENISA Threat Landscape 2025`. Документ распространяется по лицензии
`CC BY 4.0`. Полный PDF и текстовая выгрузка хранятся локально и исключены из
Git ради опрятности репозитория; канонический адрес и лицензия позволяют
использовать материал шире.

## Происхождение

- Название: `ENISA Threat Landscape 2025`
- Издатель: European Union Agency for Cybersecurity (ENISA)
- Период данных: `1 июля 2024 — 30 июня 2025`
- Год издания: `2025` (октябрь 2025; правки ноября 2025)
- Маркировка обмена: `TLP:CLEAR`
- Канонический адрес:
  `https://www.enisa.europa.eu/publications/enisa-threat-landscape-2025`
- Источник файла:
  `https://www.enisa.europa.eu/sites/default/files/2025-11/ENISA%20Threat%20Landscape%202025.pdf`
- Тип источника: ежегодный аналитический отчёт о ландшафте угроз (PDF)
- Способ получения: загрузка PDF с сайта ENISA по каноническому адресу
- Дата включения в корпус: `2026-06-25`

- Лицензия: `Creative Commons Attribution 4.0 International` (`CC BY 4.0`).
- URL лицензии: `https://creativecommons.org/licenses/by/4.0/`
- Уровень уверенности: высокий (лицензия заявлена ENISA).
- Допустимое использование: воспроизведение и адаптация при атрибуции.
- Хранение полного PDF и выгрузки в Git: не требуется (документ доступен по

- Атрибуция: `knowledge/source-attribution.md`.

## Ограничения

- Это снимок ландшафта угроз за конкретный период (`2024-07`–`2025-06`);
  числовые доли и акценты быстро устаревают и не являются вечными константами.
- Отчёт ориентирован на контекст ЕС; применимость к другим регионам требует
  оговорки.
- Текстовая выгрузка получена `pdftotext -layout`; врезки, диаграммы и сноски в
  выгрузке перемешаны; при ссылке на числа нужно сверяться с PDF.
- Извлечённые утверждения нельзя считать решениями проекта без анализа влияния.

## Назначение в проекте

Источник может использоваться как первичный материал для:

- актуального ландшафта угроз: основные категории (программы-вымогатели,
  социальная инженерия и фишинг, угрозы доступности/DDoS, вредоносное ПО, угрозы
  данным, компрометация цепочки поставок, манипуляция информацией FIMI,
  хактивизм, активность связанных с государствами групп) и наблюдаемые TTP;
- обоснования выбора угроз при моделировании угроз реальными данными;
- усиления навыка `security-threat-modeling` (что рассматривать в модели угроз)
  и контекста для `security-audit`.
