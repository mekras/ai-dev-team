# Карточка источника: `SMAS`

## Идентификатор

`SMAS`

## Статус

- Состояние учёта: учтён
- Состояние получения: получен; полный PDF и текстовая выгрузка сохранены
  локально и исключены из Git
- Состояние пригодности: пригоден для дальнейшей работы с ограничениями

## Что это за источник

Локальная PDF-копия отчёта O’Reilly
`Securing Microservice APIs: Sustainable and Scalable Access Control` (Matt
McLarty, Rob Wilson, Scott Morrison), изданного в `2018` году. Отчёт предлагает
платформонезависимую модель управления доступом к веб-API в микросервисной
архитектуре.

## Почему источник относится к проекту

Источник рассматривается как методическое основание для:

- управления доступом к API в распределённых и микросервисных системах (рамка
  IAAA);
- выбора сетевых и прикладных механизмов контроля доступа (сегментация,
  TLS/mTLS, SPIFFE, API-ключи, OAuth 2.0, OpenID Connect, JWT, ABAC);
- выбора инфраструктурных механизмов (API-шлюзы, сервисные прокси, service mesh,
  PaaS, serverless);
- проектирования сквозной модели доступа через доменную иерархию доверия
  `DHARMA`.

## Что уже известно по источнику

- Отчёт описывает рамку управления доступом `IAAA` и нефункциональные качества
  решения (управляемость, производительность, удобство для разработчика).
- Глава 2 систематизирует сетевые, прикладные и инфраструктурные механизмы.
- Глава 3 вводит модель `DHARMA` (трастовые домены, механизмы доверия и доступа,
  внутренние и граничные эндпойнты, доменная иерархия) и методологию из четырёх
  шагов с платформонезависимой реализацией в три уровня.
- Отчёт ограничен HTTP-API и не охватывает не-HTTP-протоколы и контейнеры в
  целом.
