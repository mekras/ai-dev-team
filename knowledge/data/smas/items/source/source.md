# Источник: `SMAS`

## Статус

Источник принят в корпус знаний как локально сохранённая рабочая копия отчёта
O’Reilly об управлении доступом к API в микросервисной архитектуре. Открытой
лицензии на повторное использование текста нет; права принадлежат O’Reilly
Media. Полный PDF и текстовая выгрузка хранятся локально и исключены из Git.

## Происхождение

- Название:
  `Securing Microservice APIs: Sustainable and Scalable Access Control`
- Авторы: Matt McLarty, Rob Wilson, Scott Morrison
- Издатель: O’Reilly Media, Inc.
- Год издания: `2018` (первое издание, февраль 2018)
- ISBN: `978-1-492-02711-9`
- Тип материала: первичные данные (отчёт O’Reilly, формат краткой книги)
- Тип источника: локальная PDF-копия отчёта O’Reilly
- Способ получения: по решению пользователя в проект скопирована локальная
  рабочая копия отчёта с хранением файла только в игнорируемом каталоге
  `knowledge/primary/SMAS/files/`
- Дата включения в корпус: `2026-06-24`
- Отдельное замечание: отчёт подготовлен в сотрудничестве O’Reilly и CA
  Technologies; модель `DHARMA` предложена авторами как платформонезависимый
  подход.

## Ограничения

- Это краткий отчёт (около 40 содержательных страниц), а не исчерпывающее
  руководство; он явно ограничен HTTP-API и не охватывает безопасность
  не-HTTP-протоколов и контейнеров в целом.
- Текстовая выгрузка получена из PDF утилитой `pdftotext -layout`; OCR не
  применялся, прозаический текст читается без искажений символов.
- Диаграммы (например, схемы доменной иерархии DHARMA) в выгрузку попали без
  графики; при ссылке на них нужно сверяться с PDF.
- Колонтитулы и номера страниц перемешаны с текстом в порядке вёрстки.
- Часть упомянутых технологий и продуктов отражает состояние экосистемы на
  `2018` год; при применении нужно сверяться с актуальными материалами.
- Извлечённые утверждения нельзя считать решениями проекта без отдельного
  анализа влияния.

## Назначение в проекте

Источник может использоваться как первичный материал для:

- управления доступом к API в распределённых и микросервисных системах: рамка
  IAAA (идентификация, аутентификация, авторизация, аудит);
- сетевых и прикладных механизмов контроля доступа: сегментация сети, SSL/TLS,
  mTLS, SPIFFE, токены (API-ключи, OAuth 2.0, OpenID Connect, JWT, SAML),
  непрозрачные и прозрачные токены, ABAC, PDP/PEP;
- инфраструктурных механизмов: обратные прокси и API-шлюзы, сервисные прокси,
  service mesh, PaaS (Kubernetes, Cloud Foundry, AWS), serverless;
- проектирования сквозной модели доступа: доменная иерархия доверия `DHARMA`
  (трастовые домены, механизмы доверия и доступа, граничные и внутренние
  эндпойнты, методология проектирования);
- архитектурных и нефункциональных требований к решениям безопасности
  (управляемость, производительность, удобство для разработчика).
