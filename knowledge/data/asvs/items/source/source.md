# Источник: `ASVS`

## Статус

Источник принят в корпус знаний как локально сохранённая копия стандарта
`OWASP Application Security Verification Standard 4.0.3` (англ. издание).
Документ распространяется по лицензии `CC BY-SA 4.0`. Полный PDF и текстовая
выгрузка хранятся локально и исключены из Git ради опрятности репозитория;
канонический адрес и лицензия позволяют использовать материал шире.

## Происхождение

- Название: `OWASP Application Security Verification Standard 4.0.3`
- Издатель: OWASP Foundation
- Версия: `4.0.3` (англ.)
- Год издания: `2021` (октябрь 2021)
- Канонический адрес проекта:
  `https://owasp.org/www-project-application-security-verification-standard/`
- Источник файла: `https://github.com/OWASP/ASVS` (тег `v4.0.3`)
- Тип источника: открытый стандарт требований безопасности (PDF)
- Способ получения: загрузка PDF из официального репозитория OWASP
- Дата включения в корпус: `2026-06-25`

## Условия использования

- Лицензия: `Creative Commons Attribution-ShareAlike 4.0 International`
  (`CC BY-SA 4.0`).
- URL лицензии: `https://creativecommons.org/licenses/by-sa/4.0/`
- Уровень уверенности: высокий (лицензия заявлена OWASP).
- Допустимое использование: воспроизведение и адаптация при атрибуции и
  совместимой лицензии (BY-SA).
- Хранение полного PDF и выгрузки в Git: не требуется (документ доступен по
  каноническому адресу); локальное хранение выбрано ради опрятности репозитория.
- Атрибуция: `knowledge/source-attribution.md`.

## Ограничения

- Текстовая выгрузка получена `pdftotext -layout`; нумерованные таблицы
  требований и столбцы уровней/CWE в выгрузке перемешаны; при ссылке на
  конкретное требование нужно сверяться с PDF.
- Это версия `4.0.3`; существует более новая мажорная линия ASVS, требования
  которой могут отличаться; при применении сверяться с актуальной версией.
- Извлечённые утверждения нельзя считать решениями проекта без анализа влияния.

## Назначение в проекте

Источник может использоваться как первичный материал для:

- стандарта требований и верификации безопасности приложений: три уровня доверия
  (L1, L2, L3) и 14 разделов требований (V1–V14), от архитектуры и
  аутентификации до конфигурации;
- вывода проверяемых требований безопасности и критериев приёмки;
- усиления навыка `security-audit` (верификация, уровни) и связи с
  `requirements-*` (требования безопасности) и `security-threat-modeling`
  (раздел V1 «Architecture, Design and Threat Modeling»).
