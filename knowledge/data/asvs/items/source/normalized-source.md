## Структура стандарта

ASVS (Application Security Verification Standard) — открытый стандарт требований
к безопасности веб-приложений и сервисов и одновременно основа для их
верификации. Стандарт задаёт каталог проверяемых требований, сгруппированных в
разделы, и три уровня доверия.

### Уровни верификации

- Уровень 1 (L1) — минимальный; защищает от типовых уязвимостей, полностью
  проверяем «снаружи» (в т. ч. ручным пентестом и автоматикой) без доступа к
  коду; минимум для всех приложений.
- Уровень 2 (L2, «Standard») — для приложений с чувствительными данными; требует
  доступа к коду и документации; типичная цель для большинства систем.
- Уровень 3 (L3, «Advanced») — для самых критичных приложений (высокая ценность
  транзакций, безопасность жизни); требует углублённого анализа архитектуры,
  кода и тестирования.

### Разделы требований (V1–V14)

- V1 Architecture, Design and Threat Modeling — архитектура, проектирование и
  моделирование угроз.
- V2 Authentication — аутентификация.
- V3 Session Management — управление сессиями.
- V4 Access Control — управление доступом.
- V5 Validation, Sanitization and Encoding — проверка, очистка и кодирование
  ввода/вывода.
- V6 Stored Cryptography — хранимая криптография.
- V7 Error Handling and Logging — обработка ошибок и журналирование.
- V8 Data Protection — защита данных.
- V9 Communication — защита коммуникаций.
- V10 Malicious Code — защита от вредоносного кода.
- V11 Business Logic — безопасность бизнес-логики.
- V12 Files and Resources — файлы и ресурсы.
- V13 API and Web Service — API и веб-сервисы.
- V14 Configuration — конфигурация.

Каждое требование помечено применимыми уровнями (L1/L2/L3) и сопоставлено с
идентификатором слабости CWE, что связывает стандарт с таксономией уязвимостей.

## Связь с проектом

Нормализованный слой служит картой для извлечения утверждений
(`knowledge/statements/ASVS.md`) и для анализа влияния на навык `security-audit`
(верификация по уровням), связи с `requirements-*` (требования безопасности) и
`security-threat-modeling` (раздел V1).
