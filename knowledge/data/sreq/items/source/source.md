# Источник: `SREQ`

## Статус

Источник принят в корпус знаний как локально сохранённая рабочая копия книги по
разработке требований с ограничением на публикацию полного текста в Git.

## Происхождение

- Название:
  `Разработка требований к программному обеспечению. 3-е изд., дополненное`
- Авторы: Карл Вигерс, Джой Битти
- Год русского издания: `2014`
- ISBN: `978-5-7502-0433-5`, `978-5-9775-3348-5`
- Тип источника: локальная PDF-книга
- Способ получения: по решению пользователя в проект скопирована локальная
  рабочая копия книги с хранением файла только в игнорируемом каталоге
  `knowledge/primary/SREQ/files/`
- Дата включения в корпус: `2026-06-22`

## Что сохранено

Для локальной обработки получены следующие артефакты:

- паспорт: `knowledge/primary/SREQ/source.md`
- индекс артефактов: `knowledge/primary/SREQ/page-index.tsv`
- PDF: `knowledge/primary/SREQ/files/software-requirements-3rd-ru.pdf`
- текстовая выгрузка:
  `knowledge/primary/SREQ/files/software-requirements-3rd-ru.txt`

В Git сохраняются только метаданные источника, обзорный нормализованный слой и
извлечённые утверждения без крупных дословных фрагментов книги.

## Физические характеристики

- Объём PDF: `8603950` байт
- Объём текстовой выгрузки: `3206460` байт
- Число страниц: `737`
- Заголовок PDF: `Разработка требований к программному обсеспечению`
- Автор в метаданных PDF: `Джой Битти`

## Условия использования

- Открытая лицензия на повторное использование полного текста не установлена.
- В выходных страницах книги указаны правообладатели перевода и указано, что
  перевод публикуется и продаётся с разрешения O’Reilly Media, Inc.
- Уровень уверенности: средний.
- Полный PDF и полная текстовая выгрузка не публикуются в Git публичного
  проекта.
- Допустимое использование в корпусе: локальный анализ, нормализация тем,
  извлечение собственных кратких утверждений и ссылок на артефакты.
- Ограничение: не публиковать полный текст и крупные дословные фрагменты без
  отдельного правового основания.
- Атрибуция и правовой статус: `knowledge/source-attribution.md`.

## Ограничения

- Это локальная рабочая копия книги, а не ссылка на издательский сайт.
- Текстовая выгрузка может содержать дефекты распознавания и артефакты переноса
  строк.
- При спорных формулировках нужно сверяться с PDF, а не только с выгрузкой.
- Извлечённые утверждения нельзя считать решениями проекта без отдельного
  анализа влияния.

## Назначение в проекте

Источник может использоваться как первичный материал для:

- практик выявления, анализа, документирования, проверки и управления
  требованиями;
- правил работы с бизнес-требованиями, пользовательскими требованиями,
  бизнес-правилами и атрибутами качества;
- развития навыков и методических документов, связанных с требованиями.
