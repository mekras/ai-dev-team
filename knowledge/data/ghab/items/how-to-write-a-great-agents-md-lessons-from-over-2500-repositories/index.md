# How to write a great agents.md (нормализованное представление)

## URL

- https://github.blog/ai-and-ml/github-copilot/
- how-to-write-a-great-agents-md-lessons-from-over-2500-repositories/

## Что это источник утверждает

- agents.md используется как определение персоналий (`persona`) для
  специализированных агентов (docs/test/security и др.).
- Важны четкий набор границ, точные команды, примеры и стиль взаимодействия.
- Основная ошибка большинства нерабочих файлов — расплывчатость
  (`You are a helpful assistant` без операционализации).
- Хорошие примеры показывают конкретную роль, команды и запрещенные
  действия.
- Выделены готовые шаблоны по ролям агентов и наборы проверяемых сценариев.

## Практические рекомендации из анализа 2,500+ репозиториев

- Ключевые команды лучше ставить ближе к началу с флагами и опциями
  (`npm test`, `npm run build`, `pytest -v` и т.п.).
- Кодовый пример важнее описания: один рабочий пример лучше нескольких
  абстрактных абзацев.
- Нужно явно обозначать, что делать нельзя (секреты, зоны риска,
  ограничения изменений).
- Для разных задач полезно иметь разные роли агентов, чем один универсальный.
- Наблюдается ценность шаблонов под задачи, а не только общих инструкций.
