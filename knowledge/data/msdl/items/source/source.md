# Источник: `MSDL`

## Статус

Источник принят в корпус знаний как `external_reference`: корпус хранит локатор,
условия использования, нормализованный обзор и краткие утверждения, но не хранит
копию материалов. `Microsoft Security Development Lifecycle` — проприетарные
материалы; копия не делается, материал получается из первоисточника по запросу.

## Происхождение

- Название: `Microsoft Security Development Lifecycle (SDL)`
- Издатель: Microsoft
- Канонический адрес: `https://www.microsoft.com/en-us/securityengineering/sdl`
- Практики: `https://www.microsoft.com/en-us/securityengineering/sdl/practices`
- Тип источника: методология безопасной разработки ПО (набор практик)
- Дата включения в корпус: `2026-06-25`

## Стратегия источника

- `storage_strategy: external_reference`
- `retrieval_mode: on_demand`
- `copy_policy: external_reference`
- `requires_access_check: true` (проприетарные материалы; проверять условия
  перед переносом и не публиковать крупные дословные фрагменты)

## Что сохранено в корпусе

- паспорт: `knowledge/primary/MSDL/source.md`
- нормализованный обзор: `knowledge/normalized/MSDL/source.md`
- утверждения: `knowledge/statements/MSDL.md`
- учёт: `knowledge/inventory/MSDL.md`
- локальная копия не хранится

## Условия использования

- Лицензионный статус: материалы Microsoft защищены авторским правом; открытой
  лицензии на повторное использование полного текста нет.
- Уровень уверенности: высокий по факту проприетарности.
- Допустимое использование: краткие сведения и ссылки; без публикации крупных
  дословных фрагментов.
- Атрибуция: `knowledge/source-attribution.md`.

## Ограничения

- Материалы обновляются; набор практик и формулировки меняются — сверяться с
  актуальной страницей.
- Тема в значительной мере покрыта корпусом (`TMDS` — моделирование угроз,
  `SSDF` — практики ЖЦ, `SAMM` — зрелость); MSDL — авторитетный отраслевой
  пример SDL.

## Назначение в проекте

Источник используется как отраслевой пример безопасного ЖЦ для навыков
`security-audit` и `security-threat-modeling` (практики, моделирование угроз,
обязательные действия по этапам), с опорой на уже учтённые `SSDF`/`SAMM`.
