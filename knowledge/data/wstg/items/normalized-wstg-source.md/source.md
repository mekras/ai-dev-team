## Структура руководства

WSTG — открытое руководство по тестированию безопасности веб-приложений и
сервисов. Оно описывает методологию (как встроить тестирование в жизненный цикл)
и каталог тестов, сгруппированных по категориям с устойчивыми идентификаторами
вида `WSTG-<КАТЕГОРИЯ>-<номер>`.

### Категории тестирования

- Information Gathering — сбор информации.
- Configuration and Deployment Management — конфигурация и развёртывание.
- Identity Management — управление идентичностью.
- Authentication — аутентификация.
- Authorization — авторизация.
- Session Management — управление сессиями.
- Input Validation — проверка ввода (инъекции, XSS и др.).
- Error Handling — обработка ошибок.
- Cryptography — криптография.
- Business Logic Testing — тестирование бизнес-логики.
- Client-side Testing — клиентская сторона.
- API Testing — тестирование API.

## Связь с проектом

Обзор служит основанием для навыка `security-audit` (методики и категории
тестирования веб-безопасности, пентест) и `security-tooling` (классы средств).
Конкретные тесты получать из первоисточника по идентификатору `WSTG-*`.
