  локально и исключены из Git

## Что это за источник

Локальная PDF-книга `API` (`The API Book`) Сергея Константинова о
проектировании, развитии и продуктовом сопровождении программных интерфейсов.
Книга распространяется автором по открытой лицензии `CC BY-NC 4.0`.

## Почему источник относится к проекту

Источник рассматривается как методическое основание для:

- проектирования API сверху вниз (область применения, уровни абстракции, области
  ответственности, конечные интерфейсы);
- критериев качества API и предотвращения оверинжиниринга;
- обратной совместимости как временно́й характеристики качества: слабая и сильная
  связность, версионирование, расширение через абстрагирование;
- понимания API как продукта: аудитория, бизнес-модели, KPI, документация,
  поддержка, тестовая среда;
- безопасности API и борьбы с фродом.

## Что уже известно по источнику

- Книга состоит из введения и трёх разделов: проектирование API, обратная
  совместимость, API как продукт; автор считает продуктовый раздел самым важным.
- Центральная метафора: API — формальное обязательство связывать программируемые
  контексты, а его потребители — инженеры.
- Изложение опирается на сквозной учебный пример (API заказа кофе) и движется
  сверху вниз — от смысла к конечным интерфейсам.
- Книга непрерывно развивается в репозитории автора; локальная копия — снимок на
  дату сборки PDF (`2023-04-09`).
