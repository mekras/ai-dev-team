# Источник: `APIB`

## Статус

Источник принят в корпус знаний как локально сохранённая рабочая копия книги об
проектировании и развитии API. Книга распространяется автором по открытой
лицензии `CC BY-NC 4.0`; полный текст хранится локально по решению о размере
репозитория, а не из-за ограничения лицензии.

## Ограничения

- Это локальная рабочая копия книги, а не каноническая версия; книга непрерывно
  развивается в репозитории автора, поэтому копия — снимок на дату сборки PDF.
- Текстовая выгрузка получена из PDF утилитой `pdftotext -layout`; OCR не
  применялся, прозаический текст читается без искажений символов.
- Иллюстрации, диаграммы и блоки кода в текстовой выгрузке представлены частично
  или с искажением порядка; при ссылке на них нужно сверяться с PDF.
- Колонтитулы и номера страниц перемешаны с текстом в порядке вёрстки.
- Извлечённые утверждения нельзя считать решениями проекта без отдельного
  анализа влияния.

## Назначение в проекте

Источник может использоваться как первичный материал для:

- проектирования API сверху вниз: область применения, разделение уровней
  абстракции, разграничение областей ответственности, описание конечных
  интерфейсов;
- критериев качества API (очевидность, читаемость, консистентность) и рисков
  оверинжиниринга;
- обеспечения обратной совместимости: слабая и сильная связность,
  версионирование, расширение через абстрагирование, изоляция зависимостей;
- понимания API как продукта: аудитория, бизнес-модели, KPI, документация,
  поддержка, тестовая среда, управление ожиданиями;
- безопасности API: идентификация потребителей, борьба с фродом, тиражирование
  уязвимости на потребителей;
- развития навыков и документов, связанных с архитектурой, требованиями,
  технической документацией и управлением проектом.
