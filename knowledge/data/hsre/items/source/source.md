# Источник: `HSRE`

## Статус

Источник принят в корпус знаний как `external_reference`: корпус хранит локатор,
условия доступа и (на отдельном шаге) нормализованный обзор и краткие
утверждения, но не хранит копию книги.
`Handbook of Software Reliability Engineering` — большой свод, выложенный по
главам на сайте редактора; локальная копия не делается, релевантная глава
получается из первоисточника по запросу.

## Происхождение

- Название: `Handbook of Software Reliability Engineering`
- Редактор: Michael R. Lyu
- Издатель: IEEE Computer Society Press; McGraw-Hill
- Год издания: `1996`
- ISBN: `0-07-039400-8`
- Канонический адрес: `https://www.cse.cuhk.edu.hk/~lyu/book/reliability/`
- Тип источника: книга, выложенная по главам (HTML и PDF)
- Способ получения: точечное получение релевантной главы из первоисточника
- Дата включения в корпус: `2026-06-26`

## Стратегия источника

- `storage_strategy: external_reference`
- `retrieval_mode: on_demand` (получать релевантную главу из первоисточника)
- `copy_policy: external_reference` (копия в корпусе не хранится)
- `requires_access_check: true` (доступ может измениться)
- `allowed_use`: точечная проверка со ссылкой на первоисточник; краткие
  утверждения без крупных дословных фрагментов

## Что сохранено в корпусе

- паспорт: `knowledge/data/hsre/items/source/source.md`
- учёт: `knowledge/data/hsre/source.yml`
- локальная копия книги не хранится
- нормализованный обзор и утверждения будут добавлены на отдельном шаге

## Условия использования

- Лицензия: содержимое выложено автором и издателем для свободного использования
  на сайте редактора; отдельная открытая лицензия не указана.
- Уровень уверенности: средний.
- Ограничение: не публиковать крупные дословные фрагменты без правового
  основания.
- Атрибуция и правовой статус: `knowledge/source-attribution.md`.

## Ограничения

- Копия не хранится; перед применением сверяться с доступностью глав по адресу.
- Книга издана в `1996` году; часть инструментов и данных устарела, понятийная и
  методическая база сохраняет ценность.
- Краткие утверждения нельзя считать решениями проекта без отдельного анализа
  влияния.

## Назначение в проекте

Источник может использоваться как методическое основание для:

- моделей роста надёжности ПО и операционных профилей;
- измерения надёжности, тестирования на надёжность и отказоустойчивого ПО;
- связи требований надёжности с проверяемыми моделями после анализа влияния.
