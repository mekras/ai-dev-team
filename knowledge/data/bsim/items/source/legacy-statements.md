## Назначение

Утверждения фиксируют назначение и структуру `BSIMM` для навыка
`security-audit`. Источник — `external_reference`, проприетарный; переносятся
краткие сведения без дословных фрагментов. Статус — `ready_for_review`.

### BSIM-001. Назначение модели

- Утверждение: BSIMM — модель зрелости, построенная на наблюдении того, что
  реально делают организации в программах безопасности ПО; это описательный
  ориентир для сравнения, а не предписывающий стандарт.
- Фрагмент источника: обзор BSIMM.
- Область применения: специалист по безопасности, project-manager.

### BSIM-002. Четыре домена и двенадцать практик

- Утверждение: BSIMM организован по четырём доменам — Governance, Intelligence,
  SSDL Touchpoints, Deployment — с двенадцатью практиками; каждая практика
  содержит наблюдаемые активности по уровням.
- Фрагмент источника: структура модели (страница издателя).
- Область применения: специалист по безопасности.

### BSIM-003. Отличие от SAMM

- Утверждение: BSIMM описывает, что организации делают (наблюдение), тогда как
  `SAMM` предписывает целевые уровни; их уместно использовать вместе: SAMM — для
  целей, BSIMM — для сравнения с практикой отрасли.
- Фрагмент источника: сопоставление моделей.
- Область применения: специалист по безопасности.

### BSIM-004. Доступ и актуальность

- Утверждение: отчёт обновляется ежегодно (на дату — `BSIMM16`, анализ практик
  111 организаций) и доступен бесплатно по регистрации; числа и тренды — снимок
  издания, сверять по каноническому адресу.
- Фрагмент источника: страница отчёта BSIMM16.
- Область применения: специалист по безопасности.
