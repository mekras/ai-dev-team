# Карточка источника: `CLCD`

## Идентификатор

`CLCD`

## Статус

- Состояние учёта: учтён
- Состояние получения: получен; полный PDF и текстовая выгрузка сохранены
  локально и исключены из Git
- Состояние пригодности: пригоден для дальнейшей работы с ограничениями

## Что это за источник

Локальная PDF-книга `Чистый код: создание, анализ и рефакторинг` Роберта
Мартина, русское издание книги `Clean Code`.

## Почему источник относится к проекту

Источник рассматривается как методическое основание для:

- роли программиста (`coder`);
- практик написания, анализа и рефакторинга кода;
- правил именования, функций, комментариев и форматирования;
- модульного тестирования, обработки ошибок, границ и классов;
- выявления запахов кода и безопасного улучшения реализации.

## Тип источника

- Тип материала: первичные данные
- Тип формы: PDF-документ
- Тип происхождения: локальная рабочая копия печатного издания

## Условия использования

- Открытая лицензия на повторное использование полного текста не установлена.
- Правообладатели перевода и издания указаны в выходных данных книги.
- Уровень уверенности: средний.
- Хранение полного PDF и полной текстовой выгрузки в Git публичного проекта: не
  допускается.
- Локальное хранение рабочего PDF и его текстовой выгрузки: допустимо в рамках
  выбранного проектом режима с исключением из Git.
- Допустимое использование: локальный анализ, обзор тем, извлечение кратких
  проверяемых утверждений и производных выводов без крупных дословных
  фрагментов.
- Атрибуция и правовой статус: `knowledge/source-attribution.md`.

## Что уже известно по источнику

- Книга содержит `17` глав и `3` приложения.
- Первая часть описывает принципы, паттерны и приёмы написания чистого кода;
  вторая содержит сценарии последовательного очищения; третья собирает запахи
  кода и эвристические правила.
- Текстовая выгрузка из PDF достаточна для первичного извлечения утверждений, но
  примеры кода и спорные места требуют сверки с PDF.

## Что можно делать дальше

- расширять набор утверждений по отдельным главам;
- использовать источник как основание для развития роли `coder`;
- готовить отдельный навык конструирования кода, если владелец проекта примет
  такое решение;
- сопоставлять выводы с источниками `CCMP` и `CLAR`.

## Что требует осторожности

- не публиковать в Git полный PDF, полную текстовую выгрузку и крупные дословные
  фрагменты;
- не переносить большие примеры кода из книги в отслеживаемый слой;
- не выдавать тезисы книги за принятые решения проекта;
- различать наблюдения из книги, интерпретации агента и локальные правила этого
  репозитория.
