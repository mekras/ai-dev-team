## Основание

- Источник учёта: `knowledge/inventory/CVSS.md`
- Первичный источник: `knowledge/primary/CVSS/source.md` (`external_reference`)
- Нормализованный слой: `knowledge/normalized/CVSS/source.md`
- Атрибуция: `knowledge/source-attribution.md`
- Дата извлечения: `2026-06-25`

## Назначение

Утверждения фиксируют структуру `CVSS v4.0` для приоритизации в навыках
`security-audit` и `security-threat-modeling`. Источник — `external_reference`.
Статус — `ready_for_review`. Атрибуция FIRST обязательна.

### CVSS-001. Назначение

- Утверждение: CVSS — открытая методика оценки технической серьёзности
  уязвимостей, принадлежащая FIRST и предоставляемая свободно при атрибуции.
- Фрагмент источника: спецификация v4.0.
- Область применения: специалист по безопасности.

### CVSS-002. Четыре группы метрик

- Утверждение: CVSS v4.0 включает группы метрик Base (постоянные
  характеристики), Threat (эксплуатируемость), Environmental (факторы
  развёртывания) и Supplemental (контекст, не влияет на балл).
- Фрагмент источника: разделы метрик v4.0.
- Область применения: специалист по безопасности.

### CVSS-003. Базовые метрики

- Утверждение: базовая группа делится на метрики эксплуатируемости (Attack
  Vector, Attack Complexity, Attack Requirements, Privileges Required, User
  Interaction) и влияния (конфиденциальность/целостность/доступность уязвимой и
  последующих систем: VC/VI/VA, SC/SI/SA).
- Фрагмент источника: базовые метрики v4.0.
- Область применения: специалист по безопасности.

### CVSS-004. Нотация и оговорка о риске

- Утверждение: балл помечается по использованным группам (`CVSS-B`, `CVSS-BT`,
  `CVSS-BE`, `CVSS-BTE`); CVSS оценивает техническую серьёзность, а не риск
  конкретной организации, поэтому приоритизация требует учёта контекста.
- Фрагмент источника: нотация и руководство по применению.
- Область применения: специалист по безопасности.
