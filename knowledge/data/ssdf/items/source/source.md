# Источник: `SSDF`

## Статус

Источник принят в корпус знаний как локально сохранённая копия документа NIST
`SP 800-218` (Secure Software Development Framework, v1.1). Документ создан
государственным агентством США и не защищён авторским правом (public domain),
поэтому ограничений на использование текста нет. Полный PDF и текстовая выгрузка
хранятся локально и исключены из Git ради опрятности репозитория; канонический
адрес позволяет повторно получить материал.

## Происхождение

- Название:
  `Secure Software Development Framework (SSDF) Version 1.1: Recommendations for Mitigating the Risk of Software Vulnerabilities`
- Идентификатор: `NIST SP 800-218`
- Авторы: Murugiah Souppaya, Karen Scarfone, Donna Dodson
- Издатель: National Institute of Standards and Technology (NIST), США
- Год издания: `2022` (февраль 2022)
- DOI: `10.6028/NIST.SP.800-218`
- Канонический адрес:
  `https://nvlpubs.nist.gov/nistpubs/specialpublications/nist.sp.800-218.pdf`
- Тип источника: официальный нормативно-методический документ (PDF)
- Способ получения: загрузка PDF с сайта NIST по каноническому адресу
- Дата включения в корпус: `2026-06-25`

## Условия использования

- Лицензия: документ NIST не защищён авторским правом в США (public domain);
  ограничений на повторное использование нет.
- Уровень уверенности: высокий.
- Допустимое использование: свободно; в корпусе хранятся метаданные,
  нормализованный слой и краткие утверждения.
- Хранение полного PDF и выгрузки в Git: не требуется (документ доступен по
  каноническому адресу); локальное хранение выбрано ради опрятности репозитория.
- Атрибуция: `knowledge/source-attribution.md`.

## Ограничения

- Текстовая выгрузка получена `pdftotext -layout`; крупные таблицы практик и
  столбцы ссылок (BSAFSS, BSIMM, OWASP и др.) в выгрузке перемешаны; при ссылке
  на конкретную задачу нужно сверяться с PDF.
- Документ описывает практики высокого уровня (что делать), а не конкретные
  инструменты или реализацию.
- Существует более новая редакция `SP 800-218 Rev. 1` (черновик) и профиль
  `SP 800-218A` для генеративного ИИ; в корпус включена финальная версия 1.1.
- Извлечённые утверждения нельзя считать решениями проекта без отдельного
  анализа влияния.

## Назначение в проекте

Источник может использоваться как первичный материал для:

- безопасного жизненного цикла разработки ПО как набора практик: подготовка
  организации (PO), защита ПО (PS), производство защищённого ПО (PW),
  реагирование на уязвимости (RV);
- вывода требований и проверок безопасности из практик высокого уровня;
- усиления навыков `security-audit` (практики и проверки), `security-tooling`
  (автоматизация цепочки сборки) и `security-threat-modeling` (моделирование
  угроз как часть PW).
