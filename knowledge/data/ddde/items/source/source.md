# Источник: `DDDE`

## Статус

Источник принят в корпус знаний как локально сохранённая рабочая копия книги по
предметно-ориентированному проектированию с ограничением на публикацию полного
текста в Git.

## Происхождение

- Название:
  `Предметно-ориентированное проектирование (DDD). Структуризация сложных программных систем`
- Автор: Эрик Эванс
- Оригинальное издание:
  `Domain-Driven Design: Tackling Complexity in the Heart of Software`, `2003`
- Издатель русского издания: Издательский дом «Вильямс», Москва
- Год русского издания: `2011`
- ISBN русского издания: `978-5-8459-1597-9`
- ISBN оригинального издания: `978-0-321-12521-7`
- Тип источника: локальная PDF-книга
- Способ получения: по решению пользователя в проект скопирована локальная
  рабочая копия книги с хранением файла только в игнорируемом каталоге
  `knowledge/primary/DDDE/files/`
- Дата включения в корпус: `2026-06-23`

## Что сохранено

Для локальной обработки получены следующие артефакты:

- паспорт: `knowledge/primary/DDDE/source.md`
- индекс артефактов: `knowledge/primary/DDDE/page-index.tsv`
- PDF: `knowledge/primary/DDDE/files/ddd-evans-ru.pdf`
- текстовая выгрузка: `knowledge/primary/DDDE/files/ddd-evans-ru.txt`

В Git сохраняются только метаданные источника; обзорный нормализованный слой и
извлечённые утверждения добавляются на следующих шагах без крупных дословных
фрагментов книги.

## Физические характеристики

- Объём PDF: `22728322` байт
- Объём текстовой выгрузки: `2001013` байт
- Число страниц: `444`
- Создатель PDF: `Adobe Acrobat 9.0`
- Производитель PDF: `Adobe Acrobat 9.0 Paper Capture Plug-in with ClearScan`
- Поля «Заголовок» и «Автор» в метаданных PDF не заполнены.

## Условия использования

- Открытая лицензия на повторное использование полного текста не установлена.
- Правообладатели перевода и издания указаны в выходных данных книги; издание
  выпущено Издательским домом «Вильямс».
- Уровень уверенности: средний.
- Полный PDF и полная текстовая выгрузка не публикуются в Git публичного
  проекта.
- Допустимое использование в корпусе: локальный анализ, нормализация тем,
  извлечение собственных кратких утверждений и ссылок на артефакты.
- Ограничение: не публиковать полный текст и крупные дословные фрагменты без
  отдельного правового основания.
- Атрибуция и правовой статус: `knowledge/source-attribution.md`.

## Ограничения

- Это локальная рабочая копия книги, а не ссылка на издательский сайт.
- Текстовая выгрузка получена из OCR-слоя ClearScan и содержит дефекты
  распознавания: искажённые символы кириллицы, артефакты переноса строк и
  перемешанный порядок строк на титульных страницах.
- При спорных формулировках нужно сверяться с PDF, а не только с выгрузкой.
- Извлечённые утверждения нельзя считать решениями проекта без отдельного
  анализа влияния.

## Назначение в проекте

Источник может использоваться как первичный материал для:

- понятий предметно-ориентированного проектирования: единый язык, ограниченный
  контекст, модель предметной области, слои приложения;
- тактических шаблонов: сущности, объекты-значения, агрегаты, репозитории,
  фабрики, сервисы;
- стратегического проектирования и работы с границами модели;
- развития навыков и методических документов, связанных с проектированием модели
  и архитектурой.
