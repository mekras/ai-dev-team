# Custom instructions with AGENTS.md (нормализованное представление)

## URL

- `https://developers.openai.com/codex/guides/agents-md`

## Основные тезисы

- Источник описывает формирование цепочки инструкций, которую Codex строит
  при запуске.
- Есть два уровня источников:
  - global (`~/.codex`)
  - project (от корня репозитория до текущего каталога).
- На каждом уровне проверяются:
  - `AGENTS.override.md`
  - `AGENTS.md`
  - fallback файлы по системным именам.
- В каждом каталоге выбирается один файл, после чего файлы объединяются
  по пути сверху вниз.
- Наконец, выполняется лимит по объёму (`project_doc_max_bytes`,
  по умолчанию 32 KiB).

## Последовательность приоритетов

- Сначала берётся глобальный уровень в `~/.codex`:
  `AGENTS.override.md`, если есть, иначе `AGENTS.md`.
- Затем проектный уровень: путь от корня проекта к текущей директории.
- Для каждого каталога приоритет:
  `AGENTS.override.md` → `AGENTS.md` → fallback.
- Файлы closer-to-workdir объединяются позже и перекрывают ранние.

## Что добавить в практику проекта

- Для снижения риска переполнения лимита можно увеличить лимит инструкций
  или переносить соглашения в вложенные директории.
- Структурировать правила нужно слоями:
  глобальный уровень + проект + подкаталог.
- Проверять порядок срабатывания через запуск `codex` и наблюдение,
  какие инструкции попали в финальную подсказку.
