# Источник: `CLCD`

## Статус

Источник принят в корпус знаний как локально сохранённая рабочая копия книги о
чистом коде с ограничением на публикацию полного текста в Git.

## Происхождение

- Название: `Чистый код: создание, анализ и рефакторинг`
- Оригинальное издание: `Clean Code`
- Автор: Роберт Мартин (Robert C. Martin)
- Издатель русского издания: Издательский дом «Питер», Санкт-Петербург
- Год русского издания: `2013`
- ISBN русского издания: `978-5-496-00487-9`
- ISBN оригинального издания: `978-0132350884`
- Тип источника: локальная PDF-книга
- Способ получения: по решению пользователя в проект скопирована локальная
  рабочая копия книги с хранением файла только в игнорируемом каталоге
  `knowledge/primary/CLCD/files/`
- Дата включения в корпус: `2026-06-25`

## Что сохранено

Для локальной обработки получены следующие артефакты:

- паспорт: `knowledge/primary/CLCD/source.md`
- индекс артефактов: `knowledge/primary/CLCD/page-index.tsv`
- PDF: `knowledge/primary/CLCD/files/clean-code-martin-ru.pdf`
- текстовая выгрузка: `knowledge/primary/CLCD/files/clean-code-martin-ru.txt`

В Git сохраняются только метаданные источника, обзорный нормализованный слой,
извлечённые утверждения и отчёт влияния без крупных дословных фрагментов книги.

- `storage_strategy: local_only`

- `allowed_use`: локальный анализ, обзор тем, извлечение кратких проверяемых
  утверждений и производных выводов без крупных дословных фрагментов

## Условия использования

- Открытая лицензия на повторное использование полного текста не установлена.
- В выходных данных указано, что права на издание получены по соглашению с
  Prentice Hall, Inc.; все права защищены.
- Полный PDF и полная текстовая выгрузка не публикуются в Git публичного
  проекта.
- Допустимое использование в корпусе: локальный анализ, нормализация тем,
  извлечение собственных кратких утверждений и ссылок на артефакты.
- Ограничение: не публиковать полный текст и крупные дословные фрагменты без
  отдельного правового основания.
- Атрибуция и правовой статус: `knowledge/source-attribution.md`.

## Ограничения

- Это локальная рабочая копия книги, а не ссылка на издательский сайт.
- Текстовая выгрузка может содержать дефекты извлечения, переносов,
  форматирования примеров кода и распознавания символов.
- При спорных формулировках и примерах кода нужно сверяться с PDF, а не только с
  выгрузкой.
- Извлечённые утверждения нельзя считать решениями проекта без отдельного
  анализа влияния и решения владельца проекта.

## Назначение в проекте

Источник может использоваться как первичный материал для:

- роли программиста (`coder`) и практик работы с кодом;
- правил именования, функций, комментариев, форматирования и обработки ошибок;
- практик модульного тестирования, границ, классов, систем и многопоточности;
- рефакторинга, последовательного очищения, запахов кода и эвристических правил;
- проверки, что изменения в коде остаются читаемыми, сопровождаемыми и покрытыми
  релевантными проверками.
