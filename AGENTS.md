# Правила для агентов

Этот файл задаёт локальные правила для агентов ИИ, которые помогают
разрабатывать **сам продукт**.

Проект использует `ai-dev-team@stable`.

## Видимость проекта

Этот проект является публичным.

## Общие правила

- При написании русских текстов используй навык `ru-lang`.
- При работе с исходными кодами и VCS используй навык `ru-dev`.
- Формат файлов в проекте должен соблюдать требования файла `.editorconfig`.
- Для файлов Markdown соблюдай длину строки не более 80 символов. Перед записью
  файла применяй `prettier` для автоматического форматирования.
- Файлы отчётов создавай только если это явно требует навык или пользователь.

## Правила изменения репозитория

1. Перед созданием или изменением содержательного файла сначала покажи
   предлагаемое содержимое человеку-заказчику.
2. Записывай файл только после явного подтверждения.
3. Сохраняй только утверждённый текст.
4. Не добавляй разделы «на будущее», списки ролей, структуру проекта или рабочие
   гипотезы без отдельного утверждения.
5. Если нужно предложить развитие проекта, делай это в ответе, а не в файлах
   репозитория.
6. Если файл уже существует, сначала прочитай его текущее содержимое.

## Ограничения

Не смешивай переносимое ядро продукта из `team/` с локальными особенностями
этого репозитория.

Не переписывай смысл документов ради стилистического улучшения.

Не создавай крупную структуру файлов без предварительного обоснования через
проблему, риск и проверяемую пользу.

Если задача выглядит неподходящей для ИИ или недостаточно определённой, сначала
укажи риск и предложи безопасный следующий шаг.
