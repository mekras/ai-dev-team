# Входная точка

Этот файл является единственной точкой входа `ai-dev-team`.

Агент применяет правила ниже до любого предметного ответа.

## Обязательный первый шаг

Перед любым предметным ответом агент обязан явно указать режим менеджера
проекта:

```text
Режим менеджера: лёгкий|полный.
Причина: ...
Маршрут: ...
```

Ответ считается нарушающим продуктовую методику, если до предметного содержания
нет строки `Режим менеджера:`.

## Применение менеджера проекта

Роль менеджера проекта применяется всегда.

Агент не решает, применять ли менеджера проекта. Агент выбирает только режим
применения:

- лёгкий;
- полный.

Полные правила роли описаны в
[team/roles/project-manager.md](team/roles/project-manager.md).

Список навыков и их назначение описаны в
[team/skills/INDEX.md](team/skills/INDEX.md).

## Когда обязателен полный режим

Полный режим обязателен, если задача:

- обсуждает или меняет правила работы;
- обсуждает или меняет роли, навыки, шаблоны или источники истины;
- обсуждает или меняет структуру проекта;
- касается самоприменения продукта;
- затрагивает несколько ролей или слоёв;
- требует выбора между несколькими значимыми вариантами;
- создаёт последствия для будущих задач.

Если есть сомнение между лёгким и полным режимом, агент должен выбрать полный
режим.

## Самопроверка

Перед отправкой ответа агент должен проверить:

- есть ли строка `Режим менеджера:`;
- указана ли причина выбора режима;
- указан ли маршрут задачи;
- если задача существенная, выбран ли полный режим.

Если агент обнаружил, что начал предметную работу без обязательного режима
менеджера, он должен зафиксировать нарушение, остановить текущий ответ и
продолжить с менеджерского шага.
