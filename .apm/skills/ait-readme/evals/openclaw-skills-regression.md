# Проверка: README локальной коллекции навыков

## Ситуация

Агенту поручили исправить главный `README.md` публичного репозитория
`openclaw-skills`. Проект содержит локальные навыки для OpenClaw, корпус знаний
и зависимость от общего набора правил `ai-dev-team`.

Предыдущий результат показал несколько связанных ошибок:

- заголовок был взят из slug репозитория: `# openclaw-skills`;
- русская проза содержала гибридные формы вроде `APM-пакет` и `OpenClaw skills`;
- README был привязан к цели `codex`, хотя проект не является документацией
  только для Codex;
- первый экран описывал не назначение проекта, а обслуживание репозитория:
  установить зависимости из файла блокировки, пересобрать инструкции и проверить
  рабочее дерево.

## Ожидаемое поведение

Агент должен:

- подобрать человеческий заголовок, например «Навыки для OpenClaw», если имя
  репозитория является slug, а не собственным названием проекта;
- описать назначение проекта через пользовательскую ценность навыков для
  OpenClaw;
- перестроить технические гибриды по правилам `ru-lang`: «пакет APM», «навыки
  OpenClaw», «файл блокировки»;
- отделить пользовательский сценарий установки или применения коллекции от
  локального сопровождения репозитория;
- переносить команды `apm install --frozen`, `apm compile --target codex` и
  `git status --short` в раздел разработки только при подтверждённом сценарии
  сопровождающего;
- не делать Codex единственной точкой входа, если проект и APM-пакет не
  ограничены только этим клиентом.

## Ошибочное поведение

Агент не должен:

- оставлять заголовок `# openclaw-skills` как основной человекочитаемый
  заголовок;
- начинать README с описания состава репозитория, зависимости на `ai-dev-team`,
  файла блокировки или сгенерированных инструкций;
- объявлять основным первым шагом сопровождающего пересборку инструкций для
  `codex`;
- использовать формулировки `APM-пакет`, `APM-проект`, `OpenClaw skills` или
  `lockfile` в русской прозе;
- выдавать список каталогов `skills/`, `knowledge/`, `apm.yml` и `apm.lock.yaml`
  за объяснение пользы проекта.

## Критерий прохождения

Новый читатель за первую минуту понимает:

- что это навыки для OpenClaw;
- зачем устанавливать или использовать коллекцию;
- какой первый пользовательский шаг выполнить;
- где начинается отдельный сценарий локального сопровождения;
- что проект не сводится к пересборке инструкций для одного клиентского
  инструмента.
