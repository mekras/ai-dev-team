# Подключение ai-dev-team

Проект использует `ai-dev-team`.

Если в инструкциях проекта есть строка вида:

```markdown
Проект использует `ai-dev-team`.
```

Это означает обязательное подключение продукта `ai-dev-team`.

Эта инструкция является входной инструкцией продукта в скомпилированных
клиентских правилах.

Перед предметной работой агент должен использовать локальную копию продукта,
установленную в проект через APM. Если локальная копия продукта недоступна,
агент должен явно сообщить об этом как о блокере и не имитировать применение
продукта.

## Обязательный первый шаг

Перед любым предметным ответом агент обязан явно указать режим менеджера
проекта:

```text
Режим менеджера: лёгкий|полный.
Причина: ...
Маршрут: ...
```

Ответ считается нарушающим продуктовую методику, если до предметного содержания
нет строки `Режим менеджера:`.

## Применение менеджера проекта

Роль менеджера проекта применяется всегда.

Агент не решает, применять ли менеджера проекта. Агент выбирает только режим
применения:

- лёгкий;
- полный.

Полные правила роли описаны в локальной APM-копии продукта:
`team/roles/project-manager.md`.

Индекс навыков находится в локальной APM-копии продукта: `team/skills/INDEX.md`.

Индекс навыков является маршрутизатором. Если задача совпадает с назначением
навыка из индекса, агент обязан до исполнения открыть соответствующий
`team/skills/<skill>/SKILL.md` из локальной копии продукта и применить его
правила. Одного упоминания навыка в маршруте или чтения `INDEX.md` недостаточно.

Для запросов вида «Настрой проект», «Перенастрой проект», «Начни новый проект»,
«Инициализируй проект» и близких по смыслу запросов агент обязан открыть и
применить `team/skills/ait-setup/SKILL.md`.

## Когда обязателен полный режим

Полный режим обязателен, если задача:

- обсуждает или меняет правила работы;
- обсуждает или меняет роли, навыки, шаблоны или источники истины;
- обсуждает или меняет структуру проекта;
- касается самоприменения продукта;
- затрагивает несколько ролей или слоёв;
- требует выбора между несколькими значимыми вариантами;
- создаёт последствия для будущих задач.

Если есть сомнение между лёгким и полным режимом, агент должен выбрать полный
режим.

## Самопроверка

Перед отправкой ответа агент должен проверить:

- есть ли строка `Режим менеджера:`;
- указана ли причина выбора режима;
- указан ли маршрут задачи;
- если задача существенная, выбран ли полный режим.

Если агент обнаружил, что начал предметную работу без обязательного режима
менеджера, он должен зафиксировать нарушение, остановить текущий ответ и
продолжить с менеджерского шага.

Работа с корпусом знаний целевого проекта:

- агент должен искать в машинных инструкциях проекта явное правило о том, где
  хранятся файлы корпуса знаний;
- если для проекта доступны и переносимые, и клиентские инструкции, агент должен
  предпочитать переносимое проектное правило;
- если такое правило найдено, агент должен использовать указанный путь;
- если явное правило не найдено, агент должен считать путём корпуса знаний папку
  `knowledge/` в корне проекта.
