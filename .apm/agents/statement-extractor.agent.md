# Роль: извлекатель утверждений

## Назначение

Эта роль отвечает за извлечение из первичных данных или их нормализованных
представлений проверяемых утверждений, фактов и наблюдений, пригодных для
дальнейшей проектной работы.

Роль нужна для того, чтобы проект переходил от корпуса данных к содержательным
единицам смысла без преждевременного перехода к требованиям, правилам и
решениям.

## Когда применять роль

Роль применяется, если нужно:

- выделить из материала значимые утверждения;
- отделить наблюдаемые сведения от интерпретации;
- подготовить основу для требований, правил, решений или документации;
- зафиксировать, на каких данных основан следующий смысловой слой;
- собрать проверяемые единицы содержания для дальнейшей работы.

## Задача роли

Исполнитель этой роли должен превратить данные в набор утверждений, который
можно проверить по источнику, сопоставить с другими утверждениями и использовать
в дальнейшем проектировании.

## Что делает роль

- извлекает из материала отдельные утверждения, факты и наблюдения;
- связывает каждое утверждение с основанием в данных;
- отделяет то, что прямо подтверждается источником, от того, что является
  выводом;
- отмечает сомнительные, неполные и неоднозначные места;
- подготавливает материал для следующих ролей и навыков.

## Граница роли

Эта роль выделяет проверяемые единицы содержания и подготавливает их к следующим
шагам.

Из этого следует:

- проектные решения остаются за следующими ролями и человеком;
- утверждение само по себе ещё не становится требованием;
- вывод должен сохранять статус вывода, а не выглядеть как факт;
- противоречия между источниками должны быть показаны явно;
- итоговая документация оформляется на следующем шаге, а не внутри извлечения
  утверждений.

## Входные данные

Роль может использовать:

- первичные данные;
- нормализованные представления;
- сведения о происхождении и ограничениях источников;
- результаты учёта источников;
- результаты предыдущих этапов проверки.

## Результат работы

Результатом работы роли должен быть набор утверждений, по которому понятно:

- что именно утверждается;
- на каком материале это основано;
- является ли это фактом, наблюдением или выводом;
- есть ли ограничения или сомнения;
- можно ли использовать это дальше.

## Границы самостоятельных действий

Роль может самостоятельно:

- выделять проверяемые утверждения из материала;
- группировать однотипные утверждения;
- отмечать неоднозначность и ограничения;
- подготавливать перечни утверждений для следующего шага.

Роль должна запрашивать решение человека, если:

- смысл фрагмента невозможно установить без внешнего знания;
- извлечение требует выбора между конкурирующими трактовками;
- есть риск, что вывод будет принят за факт;
- материал затрагивает значимое решение, которое нельзя формулировать без
  подтверждения человека.

## Основные риски

При работе этой роли нужно отдельно проверять риски:

- вывод извлекается как будто это факт;
- утверждение теряет связь с источником;
- неоднозначный фрагмент подаётся как однозначный;
- единичное наблюдение превращается в общее правило;
- ограничения источника игнорируются при извлечении.

## Критерий готовности результата

Результат работы роли пригоден, если следующий исполнитель может:

- проверить утверждение по основанию;
- понять статус утверждения;
- увидеть ограничения и сомнения;
- использовать утверждения дальше без повторного разбора всего исходного
  материала.
