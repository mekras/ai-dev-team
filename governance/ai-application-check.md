# Проверка применения ИИ

Этот документ задаёт минимальную проверку перед существенным использованием ИИ в проекте.

Проверка нужна не для того, чтобы запретить применение ИИ, а для того, чтобы убедиться: задача подходит для ИИ, у агента достаточно контекста, выбран правильный способ работы, результат можно проверить, а ответственность человека не подменяется автоматическим выводом агента.

## Когда применять проверку

Проверку нужно применять перед задачами, в которых ИИ:

- создаёт или меняет правила проекта;
- предлагает структуру, роли, навыки или шаблоны;
- анализирует требования, архитектуру, документацию или проектные решения;
- вносит изменения в репозиторий;
- готовит материалы, которые могут стать источником истины;
- действует с неполным, устаревшим или неоднозначным контекстом;
- предлагает решение, которое человек может принять без дополнительной проверки.

Для простых редакторских, справочных и черновых задач проверка может применяться сокращённо.

## Основные вопросы

Перед существенным применением ИИ нужно ответить на вопросы:

1. Какую задачу предполагается поручить ИИ?
2. Почему эта задача подходит для выполнения с помощью ИИ?
3. Какие исходные данные, источники и контекст нужны агенту?
4. Какие роли, навыки, правила или процедуры должны применяться?
5. Что ИИ может сделать самостоятельно?
6. Что требует решения или проверки человека?
7. Какой результат должен быть получен?
8. Как будет проверяться результат?
9. Какие риски создаёт применение ИИ в этой задаче?
10. Что нельзя принимать без подтверждения человека?

Если на ключевые вопросы нельзя ответить, задачу нужно уточнить, разделить на части или отложить до появления достаточного контекста.

## Типовые риски

При применении ИИ нужно отдельно проверять следующие риски:

- ИИ решает не ту задачу, потому что цель сформулирована неясно;
- ИИ подменяет проблему удобным шаблонным действием;
- ИИ работает с недостаточным или устаревшим контекстом;
- ИИ смешивает факты, допущения, выводы и принятые решения;
- ИИ предлагает переносимое правило на основе локального случая;
- ИИ создаёт новую роль, навык или шаблон без повторяемой потребности;
- ИИ увеличивает сложность каркаса без проверяемой пользы;
- человек принимает результат ИИ как утверждённое решение без проверки;
- результат ИИ становится документацией, но не соответствует фактическому состоянию проекта.

## Возможные решения по итогам проверки

По итогам проверки можно принять одно из решений:

- выполнить задачу с помощью ИИ в предложенном виде;
- уточнить цель или ожидаемый результат;
- собрать недостающие данные и повторить проверку;
- выбрать другую роль, навык или процедуру;
- разделить задачу на аналитическую, проектную и исполнительскую части;
- ограничить самостоятельные действия ИИ;
- передать решение человеку;
- отказаться от применения ИИ для этой задачи.

## Сокращённая форма

Для текущей работы над каркасом можно использовать короткую форму:

```text
Задача для ИИ:
Почему подходит для ИИ:
Нужный контекст:
Нужные роли/навыки/правила:
Что ИИ может сделать сам:
Что проверяет человек:
Риск неправильного применения ИИ:
Как проверяем результат:
```

## Применение к самому каркасу

При разработке этого каркаса проверка обязательна перед добавлением новых слоёв, правил, ролей, навыков, шаблонов, механизмов подключения и документов, которые могут стать источником истины.

Если изменение нельзя обосновать через проблему применения ИИ, повторяемую потребность, закрываемый риск или проверяемую пользу, его не следует добавлять в каркас.
